# 現在地ハザードマップ表示の設計（②方式）

> 関連 Issue: #22
> 対象: 訪日外国人が現在地（観光地）のハザードマップを確認できる機能

## 1. 背景・目的

HANDOFF.md のサービスコンセプト #3「現在地ハザードリスク情報」を、地図 UI として具体化する。災害アラート（#1）に加え、利用者が**自分のいる場所のハザード（洪水・土砂・津波）を地図で見て、危険かどうかを判定**できるようにする。

方式は机上検討で **②方式（ハイブリッド）** に確定済み:

- **オンライン**: 公式タイルを重畳して地図表示（地理院タイル＋重ねるハザードマップ）
- **オフライン**: 自前の市区町村単位 GeoJSON を turf.js で点-ポリゴン判定（観光地の通信不安定対策）

## 2. 画面構成

UI モックは `design/hazard-map/` に作成（ブラウザで開いてプレビュー可。Claude Design への同期は別途）。

| 画面/部品 | ファイル | 役割 |
|---|---|---|
| メイン画面 | `map-main.html` | 地図＋現在地ピン＋ハザード重畳＋上部に災害アラートバー＋下部にリスク要約シート |
| レイヤー切替 | `layer-toggle.html` | 洪水/土砂/津波のトグル＋不透明度 |
| リスク判定結果カード | `risk-result-card.html` | 危険/安全、想定浸水深、最寄り避難所への導線 |
| 凡例 | `legend.html` | 浸水深スケール・災害種別の色（多言語） |
| 出典表示 | `attribution-footer.html` | 二系統の出典（後述） |
| オフライン時表示 | `offline-state.html` | 地図画像なし＋判定テキスト＋現在地 |
| 言語切替 | `language-switcher.html` | 日英中(簡/繁)韓 |

## 3. 技術構成（オンライン表示）

- **地図ライブラリ: MapLibre GL JS**（OSS・API キー不要）。ブラウザ実行のため Next.js では **`ssr: false` のクライアント専用コンポーネント**として読み込む（#3 の Edge ランタイム制約とは無関係）
- **ベース地図**: 地理院タイル（淡色地図）をリアルタイム読み込み
- **ハザード重畳**: 重ねるハザードマップのラスタタイル（洪水/土砂/津波）を半透明オーバーレイ＋レイヤー切替
- **現在地**: Geolocation API でマーカー表示

## 4. オフライン判定

- 判定用データは #5 の自前ハザード **GeoJSON（東京都・災害種別×市区町村、生 GeoJSON を維持）** を使う。**PMTiles は使わない**（#24: タイル化ベクタは turf の点-ポリゴン判定に使えないため）
- PWA Service Worker で現在地周辺の市区町村ファイルをキャッシュ（#5 のキャッシュ方針に準拠）
- **turf.js の点-ポリゴン判定**（`@turf/boolean-point-in-polygon` 等の個別 import でバンドル最小化）で、現在地が各ハザード区域に含まれるかを判定し、想定浸水深などの属性を結果カードに表示

## 5. オフライン時の挙動

- ベース地図/ハザードタイルはオンライン前提（規約・技術の両面でオフライン保存しない）
- **オフライン時は地図画像を表示せず**、「地図は表示できません」の明示＋**判定テキスト（危険/安全・想定浸水深）＋現在地座標**でグレースフルに機能継続（`offline-state.html`）

## 6. 出典表示（二系統・#23 申し送り）

用途で出典元が分かれるため、両方を常時表示する（多言語画面でも維持）。

| 系統 | 出典 |
|---|---|
| オンライン表示（地図・ハザードタイル） | 「地理院タイル」「ハザードマップポータルサイト」（＋リンク） |
| オフライン判定データ（自前 GeoJSON） | 「国土数値情報」（#12 で変換した元データ） |

## 7. 多言語（#8 連動）

- 凡例・リスク判定文・UI ラベルを 日英中(簡/繁)韓 で提供
- 公式ハザードタイルは色分けゾーンで言語非依存のため、翻訳対象は自前 UI（凡例・判定文）に限られる
